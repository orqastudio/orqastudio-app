## Decision

The clippy `too_many_lines` lint is enforced at 50 lines with `-D warnings` (hard error). All functions must comply.

**Documented exceptions** are permitted when a function genuinely cannot be decomposed further without harming readability. Each exception requires:

1. `#[allow(clippy::too_many_lines)]` annotation on the function
2. A comment explaining why the function cannot be decomposed
3. An entry in this decision's "Approved Exceptions" section for audit trail

### Approved Exceptions

| Function | File | Lines | Justification |
|----------|------|-------|---------------|
| `run()` | `src/lib.rs` | ~83 | Tauri app builder — sequential `.plugin()` / `.invoke_handler()` wiring that is inherently a single registration sequence. Extracting sub-builders would scatter the app's wiring across functions with no clarity benefit. |

## Rationale

- 50-line limit prevents complexity accumulation — most functions benefit from extraction
- Hard enforcement (`-D warnings`) prevents gradual erosion
- Documented exceptions with audit trail prevent the allow-list from growing silently
- Each exception is a decision, not a pragma — it requires justification and is reviewable

## Consequences

- Every `#[allow(clippy::too_many_lines)]` in the codebase must have a corresponding entry in this decision
- New exceptions require updating this AD (creating an audit trail via git history)
- Periodic review: if exceptions accumulate (>5), reassess whether the limit should be raised
