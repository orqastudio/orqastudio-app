## Decision

`.orqa/` is the single source of truth for ALL governance artifacts. The `.claude/` directory exists only for Claude Code CLI compatibility and contains symlinks — never copies — pointing back to `.orqa/` subdirectories. Only `settings.json` and `worktrees/` remain as real files in `.claude/`.

Symlink map:
- `.claude/rules/` → `.orqa/process/rules/`
- `.claude/agents/` → `.orqa/process/agents/`
- `.claude/skills/` → `.orqa/process/skills/`
- `.claude/hooks/` → `.orqa/process/hooks/`
- `.claude/CLAUDE.md` → `.orqa/process/agents/orchestrator.md`

## Rationale

[AD-015](AD-015) established that governance artifacts use "native Claude Code format" stored in `.claude/`. As OrqaStudio evolved, the `.orqa/` directory structure was created to provide a richer organizational model (documentation/, planning/, governance/, team/) that goes beyond what `.claude/` supports. Having two directories with overlapping content caused governance divergence — agents wrote to `.claude/` while the app scanned `.orqa/`, resulting in inconsistent enforcement (see [IMPL-006](IMPL-006)).

Symlinks solve this: a single set of files in `.orqa/`, with `.claude/` providing CLI compatibility transparently.

## Consequences

- All agents MUST write to `.orqa/` paths, never `.claude/`
- The app scans `.orqa/` exclusively
- CLI tools read through symlinks and see the same content
- If symlinks break, fix the symlinks — don't create copies
