## The Short Version

1. Fork the repo and create a branch (`feat/`, `fix/`, `docs/`, `refactor/`, `test/`)
2. Make your changes following the [coding standards](docs/development/coding-standards.md)
3. Run `make check` — all checks must pass
4. Open a pull request with a clear description
