Agent grounding documents are dense, purposeful context injected at session start. They are not reference material — they do not repeat rules or list procedures. They answer: why does this work matter, what does good look like, and what goes wrong under pressure.

| Document | Grounds | Purpose |
|----------|---------|---------|
| [product-purpose.md](product-purpose.md) | Orchestrator, Planner, Writer | Mission, pillar demands, steering quality |
| [code-principles.md](code-principles.md) | Implementer, Reviewer | Architecture boundaries, coding philosophy |
| [artifact-principles.md](artifact-principles.md) | Orchestrator, Writer, Researcher | Graph discipline, artifact quality |
| [design-principles.md](design-principles.md) | Designer | Users, UX-first principle, design language |
| [research-principles.md](research-principles.md) | Researcher | Evidence standards, confidence levels, artifact output |
