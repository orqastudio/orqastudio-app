The documentation layer is written for humans, not the pipeline. It describes the current target state of the product, its architecture, and its development practices. When docs and code diverge, docs define the correct target state — code is updated to match.

Documentation is organised into five sections: Product (vision, governance, roadmap), Architecture (system design, decisions), User Interface (UX specs, component library), Development (coding standards, setup, commands), and Process (governance framework, workflows).
