# Verification Records

Verification records (`VER-NNN`) close the delivery pipeline and close the knowledge maturity loop. They prove that enforcement is working — rules are being followed, standards are being met, and delivery gates are satisfied.

## Pipeline Role

Verification is the **final stage** of both pipelines:

```
Delivery:  Idea → Research → Epic → Task → Verification
Process:   Observation → Understanding → Principle → Practice → Enforcement → Verification
```

A verification record is evidence, not assertion. It captures what was checked, how it was checked, and what the result was. If verification consistently fails for a rule or standard, the pipeline has a gap upstream — the enforcement is insufficient, the skill is unclear, or the decision is ambiguous.

Each record tracks: **what** is being verified, **how** evidence is collected (automated checks, manual review, metrics), and **when** verification runs (per-commit, per-session, per-epic, on-demand).
